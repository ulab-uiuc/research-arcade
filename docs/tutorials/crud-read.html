Read Nodes & Edges

Query and retrieve entities, traverse relationships, and filter results.

Overview

This tutorial covers querying and retrieving data from your ResearchArcade graph database. You'll learn how to read individual nodes, traverse relationships, filter results, and perform complex queries to extract meaningful insights from your research data.

Prerequisites

  • ResearchArcade installed and configured (see Getting Started)
  • Database populated with nodes and edges (see Add Nodes & Edges)
  • Basic understanding of graph queries

Reading arxiv_papers Nodes

Get All Papers

Retrieve all papers from the database:

arxiv_papers_df = research_arcade.get_all_node_features("arxiv_papers")
print(f"Total papers in database: {len(arxiv_papers_df)}")
print("\nFirst 5 papers:")
print(arxiv_papers_df.head())

Get Specific Paper by ID

Retrieve a single paper using its unique identifier:

paper_id = {"arxiv_id": "1706.03762v7"}
paper_features = research_arcade.get_node_features_by_id("arxiv_papers", paper_id)
print("Paper details:")
print(paper_features.to_dict(orient="records")[0])

Reading openreview_revisions_reviews Edges

Get All Edges

Retrieve all review edges:

openreview_revisions_reviews = research_arcade.get_all_edge_features("openreview_revisions_reviews")
print(len(openreview_revisions_reviews))

Get Neighborhood by ID

Get all edges connected to a specific node:

revision_id = {'revision_openreview_id': 'cX02yuzwWI'}
revision_review = research_arcade.get_neighborhood("openreview_revisions_reviews", revision_id)
print(revision_review.to_dict(orient="records"))

review_id = {'review_openreview_id': 'wumckDPIQ3'}
revision_review = research_arcade.get_neighborhood("openreview_revisions_reviews", review_id)
print(revision_review.to_dict(orient="records"))

Best Practices

  • Use indexed properties for filtering to improve query performance
  • Limit result sets using pagination for large queries
  • Be specific with relationship types when traversing to reduce search space
  • Cache frequently accessed nodes and relationships
  • Use projection to retrieve only needed properties
  • Consider query complexity and optimize for common access patterns
