Add Nodes & Edges

Create new entities and relationships in the graph database.

Overview

This tutorial demonstrates how to add nodes (entities) and edges (relationships) to your ResearchArcade graph database. You'll learn the basic CRUD operations for creating papers, authors, citations, and other research entities.

Prerequisites

  • ResearchArcade installed and configured (see Getting Started)
  • Active database connection
  • Basic understanding of graph database concepts

Creating a Paper Node

Add a new research paper with metadata such as title, abstract, publication date, and DOI:

from researcharcade import GraphDB

# Initialize database connection
db = GraphDB()

# Create a new paper node
paper = db.create_node(
    node_type="Paper",
    properties={
        "title": "Attention Is All You Need",
        "abstract": "The dominant sequence transduction models are based on complex recurrent or convolutional neural networks...",
        "year": 2017,
        "doi": "10.48550/arXiv.1706.03762",
        "arxiv_id": "1706.03762",
        "venue": "NeurIPS",
        "pdf_url": "https://arxiv.org/pdf/1706.03762.pdf",
        "created_at": "2024-01-15T10:30:00Z"
    }
)

print(f"Created paper with ID: {paper.id}")

Creating a Section Node

Add a section that belongs to a paper (e.g., Introduction, Methods, Results):

from researcharcade import GraphDB

# Initialize database connection
db = GraphDB()

# Create a new section node
section = db.create_node(
    node_type="Section",
    properties={
        "title": "Introduction",
        "section_number": "1",
        "content": "Full text content of the introduction section...",
        "word_count": 850,
        "order": 1
    }
)

# Link section to its parent paper
db.create_edge(
    from_node=paper_id,
    to_node=section.id,
    edge_type="HAS_SECTION",
    properties={"order": 1}
)

print(f"Created section with ID: {section.id}")

Batch Operations

For efficiency when adding multiple entities, use batch operations: 

from researcharcade import GraphDB

# Initialize database connection
db = GraphDB()

# Batch create multiple nodes
authors = db.create_nodes_batch([
    {
        "node_type": "Author",
        "properties": {"name": "Author 1", "affiliation": "University A"}
    },
    {
        "node_type": "Author",
        "properties": {"name": "Author 2", "affiliation": "University B"}
    },
    {
        "node_type": "Author",
        "properties": {"name": "Author 3", "affiliation": "University C"}
    }
])

print(f"Created {len(authors)} authors")

# Batch create edges
edges = db.create_edges_batch([
    {
        "from_node": authors[0].id,
        "to_node": paper_id,
        "edge_type": "AUTHORED",
        "properties": {"author_position": 1}
    },
    {
        "from_node": authors[1].id,
        "to_node": paper_id,
        "edge_type": "AUTHORED",
        "properties": {"author_position": 2}
    },
    {
        "from_node": authors[2].id,
        "to_node": paper_id,
        "edge_type": "AUTHORED",
        "properties": {"author_position": 3}
    }
])

print(f"Created {len(edges)} authorship edges")

Validation and Error Handling

Best practices for validating data before insertion and handling errors: 

from researcharcade import GraphDB
from researcharcade.exceptions import ValidationError, DuplicateNodeError

# Initialize database connection
db = GraphDB()

try:
    # Validate before creating
    if not paper_data.get("title"):
        raise ValidationError("Paper title is required")
    
    # Check for duplicates
    existing = db.find_nodes(
        node_type="Paper",
        filters={"doi": paper_data["doi"]}
    )
    
    if existing:
        raise DuplicateNodeError(f"Paper with DOI {paper_data['doi']} already exists")
    
    # Create the node
    paper = db.create_node(
        node_type="Paper",
        properties=paper_data
    )
    
    print(f"Successfully created paper: {paper.id}")
    
except ValidationError as e:
    print(f"Validation error: {e}")
except DuplicateNodeError as e:
    print(f"Duplicate found: {e}")
except Exception as e:
    print(f"Unexpected error: {e}")
    # Rollback transaction if needed
    db.rollback()

Best Practices

  • Always validate data before insertion to maintain data integrity
  • Use batch operations when adding multiple entities to improve performance
  • Set appropriate properties for all required fields
  • Check for existing entities to avoid duplicates
  • Use transactions when adding multiple related entities
  • Follow naming conventions for entity types and relationship types
  • Include timestamps for tracking creation and modification dates
  • Use consistent property names across similar node types
  • Add indices on frequently queried properties
  • Document custom properties and their expected data types
