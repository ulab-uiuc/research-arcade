Edit Nodes & Edges

Update entity properties and modify relationship attributes.

Overview

This tutorial demonstrates how to update existing nodes and edges in your ResearchArcade graph database. You'll learn how to modify properties, update metadata, handle versioning, and maintain data integrity when making changes to your research entities and relationships.

Prerequisites

  • Existing nodes and edges in the database

Updating arxiv_papers Nodes

Update Paper Metadata

Modify metadata for a paper:

# Update metadata for a paper
updated_paper = {
    'arxiv_id': '1706.03762v7',
    'metadata': {
        'venue': 'NeurIPS 2017',
        'pdf_url': 'https://arxiv.org/pdf/1706.03762.pdf',
        'citations': 75000,
        'influential': True
    }
}

research_arcade.update_node("arxiv_papers", node_features=updated_paper)
print("Paper updated successfully!")

Updating arxiv_authors Nodes

Update Author Homepage

Update an author's homepage URL:

updated_author = {
    'semantic_scholar_id': 'ss_ashish_vaswani',
    'homepage': 'https://ashishvaswani.com'
}

research_arcade.update_node("arxiv_authors", node_features=updated_author)
print("Author updated successfully!")

Best Practices

  • Always validate data before updating to prevent invalid states
  • Use transactions when updating related entities to maintain consistency
  • Implement versioning for critical entities to track changes over time
  • Batch updates when modifying multiple entities for better performance
  • Set appropriate timestamps for update operations
  • Handle errors gracefully and provide rollback mechanisms
  • Consider using optimistic locking to prevent concurrent update conflicts
  • Document property schemas and update patterns
