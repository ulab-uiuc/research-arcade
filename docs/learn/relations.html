Relations (Edges)

Complete reference for all relationship types connecting entities in ResearchArcade.

Overview

Relations define how entities connect to each other in ResearchArcade's knowledge graph. Each relation type represents a specific kind of connection, from authorship and citations to reviews and rebuttals. This page provides a comprehensive reference for all relation types, their cardinalities, and attributes.

Note: Relations connect entities within and across ArXiv and OpenReview datasets. See Entities for information on entity types.

Relation Categories

Structural Relations

Authorship, containment, and hierarchy

Reference Relations

Citations, references, and revisions

Review Relations

Reviews, decisions, and rebuttals

Authorship

Links papers to their authors, preserving author order and affiliation information.

Relation Details

Source: Paper

Target: Author

Cardinality: Many-to-Many

Direction: Bidirectional

Attributes

Attribute Type Required Description
paper_id string Yes Reference to paper
author_id string Yes Reference to author
position integer No Author order (1 = first author)
corresponding boolean No Is corresponding author

Citations

Paper-to-paper citation relationships showing which papers reference other papers.

Relation Details

Source: Paper (citing)

Target: Paper (cited)

Cardinality: Many-to-Many

Direction: Directed

Attributes

Attribute Type Required Description
citing_paper_id string Yes Paper making the citation
cited_paper_id string Yes Paper being cited
citation_count integer No Number of times cited in paper

Contains

Hierarchical containment relationships between papers and their components (sections, paragraphs, figures, tables).

Relation Details

Source: Paper/Section

Target: Section/Paragraph/Figure/Table

Cardinality: One-to-Many

Direction: Directed

Example Containment Hierarchies

  • Paper → Section
  • Paper → Figure
  • Paper → Table
  • Section → Paragraph
  • Section → Section (nested sections)

References

Paragraph-level citation contexts showing where and how papers are referenced in the text.

Relation Details

Source: Paragraph

Target: Paper

Cardinality: Many-to-Many

Direction: Directed

Attributes

Attribute Type Required Description
paragraph_id string Yes Paragraph containing reference
cited_paper_id string Yes Paper being referenced
context text No Citation context snippet

Reviews

Links reviews to the papers they evaluate in the peer review process.

Relation Details

Source: Review

Target: Paper

Cardinality: Many-to-One

Direction: Directed

Attributes

Attribute Type Required Description
review_id string Yes Reference to review
paper_id string Yes Paper being reviewed

Rebuts

Links author rebuttals to the reviews they address.

Relation Details

Source: Rebuttal

Target: Review

Cardinality: Many-to-One

Direction: Directed

Attributes

Attribute Type Required Description
rebuttal_id string Yes Reference to rebuttal
review_id string Yes Review being addressed

Revises

Tracks revision history, linking papers to their updated versions.

Relation Details

Source: Revision

Target: Paper

Cardinality: Many-to-One

Direction: Directed

Attributes

Attribute Type Required Description
revision_id string Yes Reference to revision
paper_id string Yes Paper being revised

Decides

Links editorial decisions to the papers they concern.

Relation Details

Source: Decision

Target: Paper

Cardinality: Many-to-One

Direction: Directed

Attributes

Attribute Type Required Description
decision_id string Yes Reference to decision
paper_id string Yes Paper receiving decision

Submitted To

Links papers to the venues (conferences, journals) where they were submitted or published.

Relation Details

Source: Paper

Target: Venue

Cardinality: Many-to-One

Direction: Directed

Attributes

Attribute Type Required Description
paper_id string Yes Reference to paper
venue_id string Yes Venue for submission
submission_date date No When paper was submitted
